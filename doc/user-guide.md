# 사용자 가이드 — 재벌 지분구조 시각화 시스템

**최종 업데이트**: 2026-05-22

---

## 시작하기

### 실행 방법

```bash
# 터미널에서 아래 명령어 실행
export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh"
cd ~/dev_ws/stockMonitoring
npm run dev      # 프론트엔드 서버 시작
```

브라우저에서 **http://localhost:5173** 접속

> 백엔드 서버 없이도 Mock 데이터로 모든 기능이 동작합니다.  
> 백엔드(실데이터)가 필요한 경우 별도 터미널에서 `npm run server` 실행

## 주요 기능 사용법

### ① 기업집단 선택 (왼쪽 상단)

- **19개 기업집단** 목록에서 원하는 그룹 클릭
- Mock 데이터: 삼성, 현대차, LG, SK, 롯데, 한화, POSCO, GS, HD현대, 신세계, CJ, 한진, 카카오, 두산, LS, 효성, 코오롱, 네이버, KT
- 그룹 변경 시 네트워크 맵, 스토리 내비, AI 분석이 모두 자동으로 업데이트됩니다

---

### ② 스토리 내비게이션 (왼쪽 하단)

총 4단계로 지배구조를 단계별 설명:

| 단계 | 내용 | 네트워크 변화 |
|------|------|------------|
| 1단계 | 총수 일가의 직접 지분 | 직접 연결선만 표시 |
| 2단계 | 지주사→계열사 연결 | 지주사 경로 하이라이트 |
| 3단계 | 전체 출자망 | 모든 연결 표시 |
| 4단계 | 순환출자 고리 | 순환 고리 점멸 |

각 단계 클릭 또는 스크롤로 이동합니다.

---

### ③ 중앙 네트워크 맵

**노드(원) 설명**:
| 색상 | 의미 |
|------|------|
| 노란색 별 ★ | 총수 일가 (개인) |
| 파란색 ● | 실질 지주사 |
| 빨간색 ● | 금융계열사 (금산분리 대상) |
| 초록색 ● | 캐시카우 (핵심 이익 창출) |
| 보라색 ● | 일반 계열사 |
| 연보라색 ● | 공익재단 |

**엣지(화살표) 설명**:
| 색상 | 의미 |
|------|------|
| 황금색 | 직접 지분 |
| 하늘색 | 계열사 간 지분 |
| 형광 초록색 (점선) | 순환출자 고리 |

**조작 방법**:
- **마우스 휠**: 확대/축소
- **드래그**: 화면 이동
- **노드 클릭**: 오른쪽 패널에 AI 분석 시작
- **엣지 더블클릭**: 지분율 수정 팝업 열기

---

### ④ 하단 버튼

#### ⚡ What-If 시뮬레이터

지분율을 임의로 변경하여 미래 영향을 예측하는 기능입니다.

1. **"⚡ What-If 시뮬레이터"** 버튼 클릭
2. 원하는 지분 관계의 슬라이더 또는 숫자 입력란에서 수치 변경
3. 변경된 항목은 황금색으로 표시됩니다
4. **"N개 변경 적용 & AI 예측"** 버튼 클릭
5. 중앙 네트워크 맵이 새 수치로 업데이트되고, AI 분석 결과가 표시됩니다
6. **전체 초기화**: 모든 수치를 원래 값으로 복원

> **엣지 더블클릭 수정**: 네트워크 맵에서 화살표를 더블클릭하면  
> 팝업창이 열리고 지분율을 직접 입력할 수 있습니다. Enter 또는 "적용" 버튼을 누르면  
> 우측 AI 분석이 자동으로 재분석됩니다.

#### ◎ Chord 다이어그램

지분 흐름을 원형으로 시각화합니다.
- "◎ Chord 다이어그램" 버튼 클릭 → 오버레이로 표시
- 호(arc) 위에 마우스를 올리면 해당 계열사 이름이 표시됩니다
- 리본 위에 마우스를 올리면 지분율이 표시됩니다

---

### ⑤ Wedge 분석 (오른쪽 상단)

노드를 클릭하면 해당 계열사에 대한 소유-지배 괴리도를 분석합니다.

- **직접 지분**: 총수 일가가 직접 보유한 %
- **실질 의결권**: 모든 간접 경로를 합산한 %
- **증폭 배수**: 실질의결권 ÷ 직접지분 (클수록 순환출자 효과가 큼)
- 폭포형 바 차트로 직접·간접·재단 경로별 기여도를 시각화

---

### ⑥ AI 분석 패널 (오른쪽 하단)

노드를 클릭하면 3가지 분석이 자동으로 시작됩니다:

| 섹션 | 내용 |
|------|------|
| 📊 Wedge 분석 | 지분 구조와 지배력 증폭 메커니즘 설명 |
| ⚠️ 소수주주 리스크 | 터널링, 대리인 비용 등 투자자 관점 위험 |
| ⚖️ 관련 규제 | 공정거래법, 금산분리법 관련 규제 현황 |

> AWS Bedrock 키가 설정되지 않은 경우 내장 분석 데이터를 표시합니다.  
> "⚡ 시뮬레이션" 태그가 표시되면 What-If 수치가 적용된 상태입니다.

## 자주 묻는 질문

**Q. 기업이 5개밖에 안 보입니다.**  
A. 백엔드 서버가 실행 중이면 MySQL 데이터를 사용합니다. `npm run server`로 백엔드를 시작하거나, 백엔드를 끄면 19개 Mock 데이터가 표시됩니다. MySQL에 그룹이 적은 경우 `npm run sync:dart`로 동기화하세요.

**Q. AI 분석이 항상 같은 내용입니다.**  
A. AWS Bedrock 키가 미설정된 상태입니다. `server/.env`에 AWS 키를 입력하면 실제 AI 분석이 동작합니다. `doc/api-key-setup.md` 참조.

**Q. npm 명령이 "not found"로 나옵니다.**  
A. NVM이 로드되지 않은 상태입니다. 아래 명령어를 먼저 실행하세요:
```bash
export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh"
```

**Q. KFTC 동기화가 400 오류입니다.**  
A. data.go.kr에서 서비스 활용신청 승인이 필요합니다. `doc/api-key-setup.md` 2번 항목의 Step 2를 완료하세요.

**Q. 네트워크 맵이 너무 작습니다.**  
A. 마우스 휠로 확대하거나 툴바의 `＋` 버튼을 사용하세요. 그래프는 기본 **130%** 로 시작하며 자동으로 축소되지 않습니다. `⊡` 버튼을 누르면 전체 노드가 화면에 맞춰집니다.
